    </th>

    <th>

    </th>

    </td>

    </th>

  </tr>

  <table class='user_case'>
  <tr>
    <th>
      Прецедент
    </th>
    <td>
      <i>Создание графика дежурств</i>
    </td>
  </tr>
  <tr>
    <th>
      Краткое описание
    </th>
    <td>
      Группа пользователей, живущих в одной комнате, в календаре указывает дату и время проведения ими уборки помещения
    </td>
  </tr>  
  <tr>
    <th>
      Действующие лица
    </th>
    <td>
      Авторизованные пользователи
    </td>
  </tr>
  <tr>
    <th>
      Предусловия
    </th>
    <td>
      Пользователи решили распределить уборку комнат
    </td>
  </tr>
  <tr>
    <th>
      Основной поток
    </th>
    <td>
      Пользователи указывают время, в которое они проведут уборку. В назначенный день приложение опросит соседей о качестве выполненной работы. В случае выполнения пользователю начисляется рейтинг
    </td>
  </tr>  
  <tr>
    <th>
      Альтернативный поток
    </th>
    <td>
      В случае невыполнения обязанностей, рейтинг с пользователя снимается
    </td>
  </tr>
  <tr>
    <th>
      Результат
    </th>
    <td>
      Комната чиста, пользователи имеют высокий рейтинг
    </td>
  </tr>
</table>

:fire:
  <table class='user_case'>
  <tr>
    <th>
      Прецедент
    </th>
    <td>
      <i>Организация очереди в ванную</i>
    </td>
  </tr>
  <tr>
    <th>
      Краткое описание
    </th>
    <td>
      Группа пользователей, живущих в одной комнате, имеющих совпадающий график посещения ванной отдают распоряжение очередью в руки приложения
    </td>
  </tr>  
  <tr>
    <th>
      Действующие лица
    </th>
    <td>
      Авторизованные пользователи
    </td>
  </tr>
  <tr>
    <th>
      Предусловия
    </th>
    <td>
      Пользователи испытывают нужду в ванной в одно и то же время (по разным причинам, например, кто-то чрезмерно долго ей пользуется)
    </td>
  </tr>
  <tr>
    <th>
      Основной поток
    </th>
    <td>
      Пользователи указывают время, в которое возникает наложение, предпочтения по продолжительности сеанса. Приложение распределяет время между пользователями. Если пользователь соблёл установленные сроки - баллы с него не снимаются
    </td>
  </tr>  
  <tr>
    <th>
      Альтернативный поток
    </th>
    <td>
      В случае нарушения очереди, рейтинг с пользователя снимается. Также может быть указаны слишком жйсткие требования (помыться 10 людям в течение 5 минут). В этом случае, приложение не сможет выполнить задачу
    </td>
  </tr>
  <tr>
    <th>
      Результат
    </th>
    <td>
      Все пользователи помыты
    </td>
  </tr>
</table>

:fire:
  <table class='user_case'>
  <tr>
    <th>
      Прецедент
    </th>
    <td>
      <i>Написание отзывов о комнатах и соседях</i>
    </td>
  </tr>
  <tr>
    <th>
      Краткое описание
    </th>
    <td>
      Пользователь может оставить оценку другому пользователю, что повлияет на рейтинг оцениваемого. Помимо других пользователей, есть возможность оставить отзыв на комнату. Отзывы будут отображаться на её странице
    </td>
  </tr>  
  <tr>
    <th>
      Действующие лица
    </th>
    <td>
      Авторизованные пользователи
    </td>
  </tr>
  <tr>
    <th>
      Предусловия
    </th>
    <td>
      Пользователь хочет оставить отзыв на соседа или соседей (например, потому что они громко шумят по ночам)
    </td>
  </tr>
  <tr>
    <th>
      Основной поток
    </th>
    <td>
      Пользователи оставляют числовую оценку по шкале от 0 до 10 для другого пользователя. В случае комнаты, помимо числовой оценки, можно оставить текстовый отзыв
    </td>
  </tr>  
  <tr>
    <th>
      Альтернативный поток
    </th>
    <td>
      В случае неадекватных оценок, с автора снимается рейтинг
    </td>
  </tr>
  <tr>
    <th>
      Результат
    </th>
    <td>
      Шумящие ночью имеют низкий рейтинг
    </td>
  </tr>
</table>

:fire:

  <tr>
    <th>
      Прецедент
    </th>
    <td>
      <i>Обмен вещами</i>
    </td>
  </tr>
  <tr>
    <th>
      Краткое описание
    </th>
    <td>
      Пользователи могут за установленную стоимость получить продукты или помощь от других пользователей
    </td>
  </tr>  
  <tr>
    <th>
      Действующие лица
    </th>
    <td>
      Авторизованные пользователи
    </td>
  </tr>
  <tr>
    <th>
      Предусловия
    </th>
    <td>
      В общежитии частно возникает желание не покупать что-либо в магазине, а одолжить или обменять у соседей
    </td>
  </tr>
  <tr>
    <th>
      Основной поток
    </th>
    <td>
      Пользователь заходит в каталог предлагаемых товаров или услуг, выбирает подходящее и при выполнении условий продающего может получить товар
    </td>
  </tr>  
  <tr>
    <th>
      Альтернативный поток
    </th>
    <td>
      Нарушение потока быть не может: пользователь идёт по чёткому сцениарию, выйти из которого он может только полностью
    </td>
  </tr>
  <tr>
    <th>
      Результат
    </th>
    <td>
      Пользователь получил нужный товар. Пользоавтель-продавец этот товар обменял на что-то более выгодное для себя
    </td>
  </tr>
</table>

    <th>

    </th>
